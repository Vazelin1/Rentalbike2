# Аренда электровелосипедов — Android-приложение

Приложение для управления арендой электровелосипедов: главная (активные аренды,
статистика, поиск, фильтры), велосипеды (статусы, ремонты, окупаемость), арендаторы
(история, проблемные клиенты), итоги по бизнесу. Данные хранятся локально (SQLite/Room),
интернет не требуется. Material Design 3, светлая/тёмная тема.

## Как получить APK и установить на телефон

Я не могу собрать APK у себя (нет доступа к Android SDK и интернету в песочнице), но
в репозитории уже настроена автоматическая сборка через **GitHub Actions** — GitHub
соберёт APK за вас бесплатно. Порядок действий:

1. **Создайте новый репозиторий на GitHub** (например, `ebike-rent`), публичный или приватный.
2. **Загрузите туда содержимое этой папки.** Проще всего через сайт GitHub: "Add file" →
   "Upload files", перетащите все файлы и папки (включая скрытые `.github` и `.gitignore`).
   Либо через git в терминале:
   ```
   cd EbikeRent
   git init
   git add .
   git commit -m "Первая версия приложения"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/ebike-rent.git
   git push -u origin main
   ```
3. Откройте вкладку **Actions** в репозитории на GitHub — сборка запустится автоматически
   (займёт 2–4 минуты). Когда сборка закончится (зелёная галочка):
   - откройте **Releases** справа на странице репозитория — там появится файл
     `app-debug.apk`, который можно скачать прямо с телефона;
   - либо откройте завершённый запуск во вкладке Actions → внизу раздел **Artifacts** →
     скачайте `ebikerent-debug-apk`.
4. **Установите на телефон:** скачайте `apk` файл через браузер телефона (ссылка на Release
   открывается прямо с Android), затем откройте файл — Android спросит разрешение
   "Установить из неизвестных источников" для вашего браузера, разрешите и установите.

После этого при каждом новом `git push` в ветку `main` GitHub будет пересобирать
свежий APK автоматически.

## Что уже реализовано

- Главная: статистика (всего/в аренде/свободно/ремонт/сегодня/просрочено), поиск по
  номеру велосипеда и ФИО, быстрые фильтры, список активных аренд с цветовой индикацией,
  карточка аренды (редактирование даты/суммы/комментария/залога, отметка возврата залога,
  фото после возврата, продление, завершение).
- Новая аренда: выбор свободного велосипеда, существующий/новый арендатор, все поля из ТЗ.
- Велосипеды: список со статусами, карточка (покупка, пробег, доход, ремонты, окупаемость/
  прибыль), фото велосипеда (маленькие миниатюры в карточке, нажатие открывает фото на весь
  экран), история ремонтов, история аренд по велосипеду.
- Арендаторы: список с поиском, подсветка проблемных клиентов, карточка с историей аренд
  и общей суммой оплат.
- Итоги: вся статистика бизнеса по ТЗ, расчёт автоматический.
- Уведомления: ежедневная фоновая проверка (WorkManager) — за N дней до конца аренды
  (N настраивается в настройках), в день окончания, ежедневно по просроченным.
- Настройки: дни предупреждения, светлая/тёмная/системная тема, резервное копирование и
  восстановление базы данных (сохраняется как файл, который можно положить в облако).
- Автосохранение всех изменений — отдельной кнопки "Сохранить" в списках нет, только там,
  где это уместно (редактирование карточки аренды).

## Упрощения в этой первой версии (можно доработать)

- Добавлена схема базы версии 2 (для фото велосипедов). Если вы уже устанавливали более
  раннюю версию приложения и обновляете её этой сборкой, локальная база данных будет
  пересоздана заново (простая стратегия для раннего этапа разработки) — резервную копию
  старых данных стоит сделать в Настройках перед обновлением, если она у вас уже была.
- Фото "перед выдачей" пока не привязаны отдельным шагом при создании аренды — вкладка
  "Фото" в карточке аренды сохраняет снимки после возврата; расширить на этап выдачи —
  дело одной похожей формы.
- Расчёт "окупаемость/прибыль" в разделе **Итоги** сделан агрегированно по всему бизнесу
  сразу (сумма покупок и ремонтов против общего дохода); на карточке **конкретного
  велосипеда** расчёт точный и по нему. Если нужно, чтобы "Итоги" считали окупаемость
  велосипед-за-велосипедом и суммировали — это несложно добавить.
- Многопользовательский режим (логины/пароли для менеджеров), облачная синхронизация,
  экспорт в Excel/PDF — архитектура (Repository поверх Room) готова для этого расширения,
  но сами эти функции ещё не реализованы, как и указано в ТЗ как "для будущих обновлений".

## Структура проекта

```
app/src/main/java/com/ebikerent/app/
  data/        — Room: таблицы, DAO, репозиторий с бизнес-логикой
  ui/          — экраны Compose (home, bikes, renters, summary, rental, settings)
  notification/ — ежедневная проверка аренд (WorkManager)
  util/        — даты, форматирование денег, резервное копирование
```

## Локальная сборка (если у вас есть Android Studio)

Откройте папку `EbikeRent` в Android Studio (Hedgehog/Iguana или новее) — она сама
подтянет зависимости и позволит запустить приложение на эмуляторе/телефоне через
обычную кнопку Run, либо собрать APK через Build → Build APK(s).
